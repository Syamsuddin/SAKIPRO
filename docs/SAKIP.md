## 1. Proses Evaluasi SAKIP

Proses evaluasi SAKIP pemerintah daerah adalah proses menilai apakah sistem manajemen kinerja pemerintah berjalan efektif, terukur, dan menghasilkan dampak nyata terhadap masyarakat.

Evaluasi tidak hanya memeriksa dokumen. Evaluasi memeriksa:

1. Kualitas perencanaan.
2. Hubungan kinerja dan anggaran.
3. Penggunaan data kinerja.
4. Efektivitas program.
5. Budaya kinerja organisasi.

Pelaksana evaluasi:

1. KemenPANRB untuk provinsi dan kabupaten/kota.
2. Inspektorat untuk evaluasi internal.
3. Pemerintah provinsi dalam pembinaan tertentu.

Dasar utama:

1. Perpres 29 Tahun 2014 tentang SAKIP.
2. PermenPANRB 88 Tahun 2021 tentang Evaluasi AKIP.

Tujuan evaluasi:

1. Mengetahui kualitas implementasi SAKIP.
2. Menilai efektivitas penggunaan anggaran.
3. Menilai ketercapaian sasaran pembangunan.
4. Memberikan rekomendasi perbaikan.
5. Menentukan predikat SAKIP daerah.

Predikat:

| Predikat | Makna Sederhana |
| --- | --- |
| AA | Kinerja sangat efektif dan berdampak tinggi. |
| A | Kinerja efektif, dokumentasi dan implementasi sudah kuat. |
| BB | Sistem sudah baik tetapi efisiensi atau kualitas outcome masih perlu ditingkatkan. |
| B | Sistem berjalan tetapi masih ada kelemahan mendasar. |
| CC | Dokumen ada tetapi implementasi belum kuat. |
| C | Banyak kelemahan dokumen dan implementasi. |
| D | Sistem belum berjalan. |

## Tahap evaluasi SAKIP

Tahap 1. Persiapan Evaluasi

Pelaksana:
Tim evaluator.

Dokumen awal yang diminta:

1. RPJMD.
2. Renstra.
3. RKPD.
4. Renja.
5. IKU.
6. Pohon kinerja.
7. Cascading.
8. PK.
9. LKjIP.
10. LHE sebelumnya.
11. Bukti tindak lanjut.

Kegiatan:

1. Menentukan ruang lingkup.
2. Menentukan OPD sampel.
3. Menyusun jadwal.
4. Menyusun LKE.

Dokumen utama evaluator:
LKE, Lembar Kerja Evaluasi.

Isi LKE:
Daftar pertanyaan dan bukti yang harus diverifikasi.

Tahap 2. Desk Evaluation

Evaluator memeriksa dokumen sebelum turun lapangan.

Yang diperiksa:

1. Konsistensi dokumen.
2. Kualitas indikator.
3. Hubungan sasaran dan program.
4. Ketepatan target.
5. Konsistensi data.

Contoh pemeriksaan:

RPJMD:
Menurunkan angka kemiskinan.

↓

Renstra Dinsos:
Meningkatkan perlindungan sosial.

↓

Program:
Bantuan sosial keluarga miskin.

Jika tidak nyambung:
Nilai turun.

Hal yang sering ditemukan:

1. Indikator output dianggap outcome.
2. Program tidak mendukung sasaran.
3. PK hanya copy Renstra.
4. LKjIP tidak sesuai PK.

Tahap 3. Evaluasi Lapangan

Metode:

1. Wawancara.
2. Observasi.
3. Uji petik data.
4. Verifikasi bukti.

Pihak yang diwawancarai:

1. Kepala daerah.
2. Sekda.
3. Kepala OPD.
4. Kasubbag perencanaan.
5. Tim SAKIP.
6. Pengelola data.

Yang diuji:

1. Apakah pimpinan memahami sasaran.
2. Apakah data indikator dipakai mengambil keputusan.
3. Apakah rapat kinerja rutin dilakukan.
4. Apakah anggaran diarahkan pada prioritas.

Contoh pertanyaan evaluator:

1. Mengapa indikator ini dipilih?
2. Apa sumber data?
3. Bagaimana menghitung capaian?
4. Apa tindak lanjut jika target gagal?

Tahap 4. Penilaian Perencanaan Kinerja

Fokus utama:
Apakah dokumen perencanaan tersusun logis dan berorientasi hasil.

Yang dinilai:

1. Kualitas tujuan.
2. Kualitas sasaran.
3. Kualitas IKU.
4. Cascading.
5. Pohon kinerja.
6. Hubungan anggaran dan sasaran.

Contoh indikator buruk:
Jumlah pelatihan guru.

Karena:
Hanya output.

Contoh indikator baik:
Persentase peningkatan kompetensi guru.

Karena:
Mengukur hasil.

Hal paling penting:
Outcome oriented.

Tahap 5. Penilaian Pengukuran Kinerja

Fokus:
Apakah daerah benar-benar mengukur capaian.

Yang dicek:

1. Monitoring berkala.
2. Dashboard kinerja.
3. Data realisasi.
4. Analisis deviasi.
5. Penggunaan data.

Evaluator mencari bukti:

1. Rapat evaluasi.
2. Notulen.
3. Instruksi pimpinan.
4. Perubahan kebijakan akibat data.

Kesalahan umum:

1. Data tidak valid.
2. Tidak ada analisis.
3. Pengukuran hanya formalitas.

Tahap 6. Penilaian Pelaporan Kinerja

Dokumen utama:
LKjIP.

Yang dinilai:

1. Kesesuaian dengan PK.
2. Analisis keberhasilan.
3. Analisis kegagalan.
4. Efisiensi anggaran.
5. Perbandingan target dan realisasi.

Evaluator tidak suka:
Narasi panjang tanpa analisis.

Yang dicari:

1. Mengapa target tercapai.
2. Mengapa target gagal.
3. Apa solusi perbaikan.

Contoh analisis baik:
Target stunting gagal karena wilayah terpencil belum terjangkau tenaga kesehatan. Tahun berikutnya dilakukan penambahan pos layanan keliling.

Tahap 7. Penilaian Evaluasi Internal

Fokus:
Apakah pemerintah daerah melakukan evaluasi mandiri.

Dokumen:

1. SK tim evaluator internal.
2. LHE internal.
3. LKE internal.
4. Rencana aksi tindak lanjut.

Yang dicari:

1. Evaluasi rutin.
2. Rekomendasi tajam.
3. Tindak lanjut nyata.

Masalah umum:

1. Evaluasi hanya administratif.
2. Tidak ada tindak lanjut.
3. Rekomendasi tidak spesifik.

Tahap 8. Penyusunan Nilai dan Predikat

Evaluator memberi skor tiap komponen.

Contoh sederhana:

| Komponen          | Nilai |
| ----------------- | ----: |
| Perencanaan       | 24/30 |
| Pengukuran        | 22/30 |
| Pelaporan         | 12/15 |
| Evaluasi internal | 18/25 |
| Total             |    76 |

Predikat:
BB.

Tahap 9. Penyusunan LHE

Dokumen:
Laporan Hasil Evaluasi.

Isi:

1. Nilai.
2. Predikat.
3. Kelebihan.
4. Kekurangan.
5. Rekomendasi.

Contoh rekomendasi:

1. Perbaiki cascading sampai level individu.
2. Gunakan indikator outcome.
3. Tingkatkan kualitas analisis LKjIP.
4. Hubungkan anggaran dengan sasaran prioritas.

Tahap 10. Tindak Lanjut

Pemerintah daerah wajib:

1. Menyusun rencana aksi.
2. Memperbaiki dokumen.
3. Memperbaiki sistem pengukuran.
4. Melaporkan tindak lanjut.

Dokumen:

1. Matriks tindak lanjut.
2. Bukti perbaikan.
3. Progress report.

Siklus evaluasi:

Perencanaan
↓
Pengukuran
↓
Pelaporan
↓
Evaluasi
↓
Rekomendasi
↓
Perbaikan
↓
Evaluasi berikutnya

Hal yang paling menentukan nilai SAKIP

1. Cascading logis.
2. Indikator outcome.
3. Kinerja dan anggaran terhubung.
4. Data valid.
5. Monitoring rutin.
6. LKjIP analitis.
7. Pimpinan aktif memakai data.
8. Tindak lanjut nyata.

Ciri pemerintah daerah dengan nilai tinggi:

1. Program sedikit tetapi fokus.
2. Sasaran jelas.
3. Dashboard aktif dipakai.
4. Anggaran berbasis prioritas.
5. Data antar OPD konsisten.
6. Evaluasi internal kuat.
7. Pimpinan memahami indikator strategis.

Ciri nilai rendah:

1. Banyak indikator output.
2. Sasaran terlalu banyak.
3. LKjIP copy paste.
4. Anggaran tidak fokus.
5. Data berbeda antar dokumen.
6. Tidak ada analisis masalah.
7. Cascading tidak nyambung.

Contoh hubungan evaluasi nyata:

Sasaran:
Penurunan kemiskinan.

↓

Program:
Pelatihan UMKM.

↓

Evaluator bertanya:
Apakah kemiskinan turun?

Jika jawabannya hanya:
“Peserta pelatihan 500 orang.”

Maka nilai rendah.

Karena evaluator ingin:
dampak terhadap masyarakat, bukan hanya aktivitas.
